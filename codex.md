# marina — project codex

Short reference for how this repo is built and run.

## commands

| Command | What it does |
|--------|----------------|
| `make server` | Serve site at http://localhost:8037 |
| `make blog` | Rebuild `public/blog/posts.json` from `public/blog/posts/*.md` |
| `make link` | Same as `make blog` (build blog / wire everything) |
| `make test` | Run pytest in `tests/` |
| `make lint` | Run `make link`, then Ruff + ESLint via pre-commit |

## pre-commit

On each commit, in order: **link** → **test** → **ruff** → **eslint**.

- Install: `pip install -r requirements-dev.txt` then `pre-commit install`
- Run all hooks: `pre-commit run --all-files`

## structure

- **public/** — static site (HTML, CSS, JS, images). Served as-is.
- **public/blog/posts/** — blog post `.md` files (YAML frontmatter: `title`, `date`).
- **public/blog/posts.json** — generated by `make blog`; consumed by blog list JS.
- **scripts/build_blog.py** — builds `posts.json` from post markdown.
- **tests/** — pytest tests for `build_blog` (parse_frontmatter, main).
- **requirements-dev.txt** — pytest, pre-commit, ruff (for local/CI).

## conventions

- New post: add `public/blog/posts/<slug>.md` with frontmatter, run `make blog`.
- Python: Ruff (no project config). JS: ESLint, see `eslint.config.js`.
- CI: `.github/workflows/ci.yml` runs pre-commit on push/PR to main/master.
